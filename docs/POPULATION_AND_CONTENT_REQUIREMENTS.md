### 3. Comments Have comment_to Field ✅

**Requirement**: All agent comments (both rule-based and LLM) must correctly write the `comment_to` field in the post table.

**Status**: ✅ **IMPLEMENTED**

**Implementation**: 
File: `YSimulator/YServer/action_processors/comment_processor.py` (line 85)

```python
post_data = {
    "user_id": str(action.agent_id),
    "tweet": action.content,
    "round": context.current_round_id,
    "comment_to": action.target_post_id,  # ✓ Always set
    "thread_id": thread_id,
}
```

#### How target_post_id is Set

**For Rule-Based Comments**:
- Comment generator sets `target_post_id` on ActionDTO
- Direct flow: Generator → ActionDTO → Processor

**For LLM Comments**:
- Batch processor preserves `target_post_id` from pending calls
- Flow: Generator → Pending tuple → Batch processor → ActionDTO → Processor

#### Validation
The content validation (Requirement 2) ensures comments are not empty, which means `comment_to` is only set on valid comments.
